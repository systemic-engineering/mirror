# Conversation Package System

**Date:** 2026-04-02
**Status:** Draft
**Authors:** Alex + Reed

## The Idea

The conversation language IS the package manager. `.conv` grammars define interfaces. The conversation runtime generates Nix flakes. Fragmentation projects source trees. Nix builds. BEAM runs.

No separate package manifest. No hand-written flakes. The grammar is the single source of truth for types, dependencies, mount points, and build configuration.

## Architecture

```
user writes .conv grammars
    ↓
conversation parse → type-check composition
    ↓
conversation generates flake.nix
    ↓
nix build → fragmentation.lib.project merges source trees
    ↓
gleam compile the projected tree
    ↓
BEAM release
```

### Core Concepts

**Core** is conversation-beam: actors, cluster, store, spectral, MCP. Always present. Not a package — the substrate.

**Package** is a mountable fragment with a `.conv` grammar that declares its types and actions. Examples: `@admin` (web dashboard), `@i18n` (natural language translation), `@ai` (model-backed projection).

**Grammar as contract:** Each package's `.conv` grammar declares what it provides (types, actions) and what it requires (types from core or other packages via `in @domain`). The conversation compiler validates that all requirements are satisfied before generating the flake.

**Lens** is a fragmentation projection that maps a package's source into the core's source tree. The lens is derived from the grammar — the package name determines the mount path.

### The DSL

When a user writes a project, they write `.conv` files. The conversation binary reads them and generates the flake:

```
grammar @myapp in @admin {
  type page = landing | settings
  action render(page)
}
```

This declares: `@myapp` extends `@admin`, which means `@admin` must be mounted. The conversation binary:
1. Parses all `.conv` files in the project
2. Resolves `in @admin` → admin package required
3. Validates admin's types are available
4. Generates `flake.nix` with admin mounted via fragmentation lens

The generated flake looks like:

```nix
{
  inputs = {
    conversation.url = "github:systemic-engineering/conversation";
    admin.url = "github:systemic-engineering/conversation-admin";
  };

  outputs = { self, conversation, admin, ... }:
    conversation.lib.beam {
      name = "myapp";
      src = ./.;
      packages = {
        admin = { src = admin; port = 4200; };
      };
    };
}
```

But the user never writes this. They write `.conv`. The runtime generates the flake.

### conversation.lib.beam

Lives in `/Users/alexwolf/dev/projects/conversation/flake.nix`. A Nix function that:

1. **Takes** a declaration: name, src, packages (with their flake inputs and options)
2. **Validates** package requirements against core capabilities
3. **Projects** via `fragmentation.lib.project` — merges core + package source trees into one tree
4. **Builds** — runs `gleam build` on the projected tree
5. **Returns** — `{ packages.default, devShells.default, devShells.agent }`

```nix
# In conversation's flake.nix
lib.beam = { name, src, packages ? {}, ... }:
  let
    # Resolve package sources
    packageSources = mapAttrs (name: cfg: cfg.src + "/src") packages;

    # Create fragmentation projection
    projected = fragmentation.lib.project {
      inherit src;
      lenses = packageSources;
    };

    # Build
    beam = gleamBuild projected;
  in {
    packages.default = beam;
    devShells.default = mkShell { buildInputs = [ beam ]; };
  };
```

### conversation.lib.package

Each package declares itself:

```nix
# conversation-admin/flake.nix (generated by conversation)
conversation.lib.package {
  name = "admin";
  grammar = ./admin.conv;  # the interface contract
  src = ./src;
}
```

The grammar file (`admin.conv`) is:

```
grammar @admin {
  type route = path | handler
  type panel = dashboard | garden | prism

  action serve(port)
  action mount(panel)
}
```

The Nix function reads the grammar, extracts the types and requirements, and exposes them for validation.

### Make Illegal State Unrepresentable

These fail at `nix eval`, not at runtime:

| Illegal state | How it fails |
|---|---|
| Mount @admin without cluster in core | Grammar requires cluster types → eval error |
| Two packages provide same module name | Lens collision detected → eval error |
| Package requires type not in core | Type resolution fails → eval error |
| Package version incompatible with core | Grammar version mismatch → eval error |
| Circular package dependency | Nix flake input cycle → eval error (Nix built-in) |

### @nix Domain

Flake generation is not special-cased. It's a domain grammar:

```
grammar @nix {
  type flake = inputs | outputs
  type input = url | follows
  type output = package | shell | lib
  type derivation = build | check | dev

  action emit(flake)
  action build(derivation)
}
```

`conversation init` compiles the project's grammars through `@nix`, and
`@nix.emit(flake)` produces `flake.nix`. Same architecture as
`@erlang.exec(apply/3)` produces BEAM modules. The domain provides content.
The runtime provides projection. `@nix` is just another domain — the one
whose output happens to be the build system.

```
conversation init          # @nix.emit(flake) from .conv files in cwd
conversation init --update # re-emit after grammar changes
```

The flake is a build artifact of the grammars — deterministic, reproducible,
content-addressable.

## The Admin Package

First package. Proves the system.

**Grammar** (`admin.conv`):
```
grammar @admin {
  type route = path | handler
  type panel = dashboard | garden | prism
  type api = state | grammar | cluster

  action serve(port)
  action mount(panel)
  action respond(api)
}
```

**Source** (Gleam): router.gleam, api.gleam, api_ffi.erl, dashboard.gleam, web.gleam, cluster.gleam, cluster_ffi.erl

**Requirements:** Core must have conversation_actor, conversation_cluster, conversation_store (validated via grammar type resolution).

**Mount:** `@admin` source merges into the core source tree at build time. Gleam compiles both as one unit.
